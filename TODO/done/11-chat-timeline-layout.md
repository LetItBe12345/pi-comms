# 阶段 11：聊天时间线与消息布局

目标：保留独立群聊 ChatView，改用完整追加式时间线和终端原生 scrollback，并修正消息左右布局。

状态：已完成。

## 已确认规则

- 初次加载最近 100 条，在线期间只追加、不截断；退出重进后重新加载最新 100 条。
- 小于 60 列视为窄屏。
- 两分钟按相邻两条消息的时间差计算，等于两分钟仍合并。
- 用户消息为纯文本，Agent 消息使用 Pi Markdown。
- 加入、离开通知只存在于在线客户端内存，不写入 SQLite，也不显示自己的加入提示。
- 旧请求晚到失败时追加系统消息；初次快照可显示已持久化的最终失败状态。
- 窗口缩放、断线重连快照、控制面板和退出确认允许完整重绘。
- CI 做文本、宽度和终端控制序列测试；40、80、120 列使用真实 Pi 手动截图，不做像素级 CI 对比。
