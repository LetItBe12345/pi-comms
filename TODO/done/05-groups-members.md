# 阶段 5：群组与成员

目标：实现群组、成员、名称和在线状态的内存模型。

## 群组任务

- [x] 创建群组。
- [x] 加入群组。
- [x] 限制一个用户同时只能加入一个群组。
- [x] 维护群组的在线连接和成员快照。

## 成员任务

- [x] 一个连接同时注册一个用户成员和一个 Agent 成员。
- [x] 为成员生成稳定的内部 ID。
- [x] 广播成员上线和离线变化。
- [x] Session 关闭时让两个成员同时离线。

## 名称任务

- [x] 加入前要求设置用户名称和 Agent 名称。
- [x] 检查同一群内所有显示名称唯一。
- [x] 检查用户名称与 Agent 名称不能重复。
- [x] 名称冲突时拒绝加入。
- [x] 校验名称长度为 1～24 个字符。
- [x] 只允许中文、英文、数字、`_` 和 `-`。
- [x] 禁止名称包含空格。
- [x] 确保名称只用于显示和 `@`，路由始终使用 ID。

## 完成条件

- [x] 两个 Session 可加入同一群组并看到四个独立成员。
- [x] 名称冲突和非法名称都会被拒绝。
- [x] 任一 Session 退出时，它的用户和 Agent 同时离线。
- [x] 离线 Agent 不会收到定向消息，失败状态会公开广播。

手动验收见 [双 Pi 群组手动测试](../../docs/manual-groups-test.md)。
