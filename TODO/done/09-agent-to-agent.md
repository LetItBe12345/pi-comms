# 阶段 9：Agent 对 Agent

目标：Agent 回答可以继续 `@` 另一个 Agent，同时限制自动通信循环。

## 任务

- [x] 只解析 Agent 最终回答开头的一个有效 `@名称`。
- [x] 公开显示原回答，并把后续路由状态附在回答下。
- [x] 新请求生成 `chainId`，后续请求沿用并增加轮数。
- [x] 支持任意数量 Agent，复用现有 FIFO 队列。
- [x] 注入发送方和目标 Agent 各自的所属用户。
- [x] 路由前检查目标、在线状态和接收权限。
- [x] 处理空正文、自身、用户、不存在目标、拒绝和执行失败。
- [x] 每 10 轮暂停，等待原发起 Pi Session 继续或结束。
- [x] `Ctrl+P` 显示待决定链；继续时增加 10 轮且沿用原链。
- [x] 将暂停决定持久化到 SQLite，并在 Broker 重启后恢复。
- [x] 增加协议、Broker、数据库、Extension 和 TUI 自动测试。
- [x] 完成 TUI 截图和三个真实 Pi Session 的 PTY 验收。

## 完成条件

- [x] 多个 Agent 可以通过公开消息连续自动通信。
- [x] 所有请求和回答保留正确的 `chainId` 与轮数。
- [x] 第 10 轮后不会继续自动注入，原发起者可继续下一段 10 轮。
- [x] 三个真实 Pi Session 完成 A → B → C → A 和第 10 轮暂停验收。
