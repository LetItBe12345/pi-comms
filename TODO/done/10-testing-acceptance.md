## 首个验收场景

分别在两个终端启动 Pi 并加载 Extension。A 发送：

```text
@B-Pi 检查当前项目的 package.json
```

- [x] A 和 B 都能看到公开消息。
- [x] 消息只进入 B 的 Pi Session。
- [x] B 忙碌时消息进入队列。
- [x] B 的最终回答只回传一次。
- [x] 回答能通过 `requestId` 对应原始请求。
- [x] A 能看到 B 的回答。
- [x] B 退出后，B 用户和 B-Pi 同时离线。
- [x] 离线目标不能收到消息。
- [x] 失败状态会公开显示。

## MVP 完成定义

- [x] 两个 Pi Session 可以稳定加入同一群组。
- [x] 用户和 Agent 都显示为独立成员。
- [x] 同一群内所有显示名称唯一。
- [x] 普通消息公开广播。
- [x] `@Agent` 消息只注入目标 Session。
- [x] Agent 最终回答公开且只回传一次。
- [x] Agent 忙碌时按顺序处理。
- [x] 三种接收权限可用。
- [x] Agent 对 Agent 每连续通信 10 轮必须暂停，未经原发起 Session 确认不得继续。
- [x] Session 生命周期和在线状态正确。
- [x] 私人 Pi 对话不会泄露到群聊。
