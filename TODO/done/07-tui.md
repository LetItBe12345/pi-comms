# 阶段 7：最小 TUI

目标：提供可以创建或加入群组、查看消息并发送文本的最小聊天界面。

## 任务

- [x] 创建 `src/tui/chat-view.ts`。
- [x] 使用 `ctx.ui.custom()`。
- [x] 使用 `@earendil-works/pi-tui`。
- [x] 注册正式命令 `/comms`。
- [x] 删除面向用户的临时 `/comms-*` 命令。
- [x] 进入 `/comms` 时复用当前 Pi Session，不创建新 Session。
- [x] 按需连接本机唯一 Broker，并在不存在或崩溃时自动启动。
- [x] 提供用户名称和 Agent 名称设置。
- [x] 提供创建入口和已有群组选择列表。
- [x] 显示群组名称和连接状态。
- [x] 显示在线用户和 Agent 成员。
- [x] 从 SQLite 加载并显示公开群聊历史。
- [x] 显示实时消息。
- [x] 复用 Pi TUI `Editor` 的多行输入和默认编辑键位。
- [x] 支持发送普通消息和 `@Agent` 消息。
- [x] 支持 `@` 自动补全。
- [x] 支持中文输入法。
- [x] 支持消息滚动和自适应布局。
- [x] 支持 Enter 发送、Shift+Enter 或 Ctrl+J 换行。
- [x] 支持 Esc 退出 TUI。
- [x] 显示发送失败状态。
- [x] 显示 Agent 忙碌、空闲和离线状态。
- [x] 退出时清理 TUI 状态，但保留当前 Pi Session 上下文。
- [x] 增加自动交互测试、真实双 Pi 测试和 TUI 截图验收。

## 本阶段不做

- 不实现鼠标操作。
- 不实现复杂富文本。

## 完成条件

- [x] 两个终端能通过 TUI 完成公开文本聊天。
- [x] 两边都能看到群名、在线成员、历史消息和实时消息。
- [x] `@Agent` 请求可完成既有的远程 Agent 闭环。
- [x] 设置、聊天、补全、状态和窄窗口截图通过视觉检查。

验收记录见 [TUI 手动验收](../../docs/manual-tui-test.md)。
