# 阶段 17：附近群组默认开放加入

目标：附近群组默认无需邀请码，只有群主在创建时明确启用邀请码，首次加入才要求输入。

## 产品规则

- 新建附近群组默认允许直接加入。
- 创建附近群组时提供“直接加入”和“使用邀请码”两项，默认选中“直接加入”。
- 已有群组保持当前状态；已有邀请码的群组继续要求邀请码。
- 开放群组和受限群组都会在首次加入后签发长期成员凭证。
- 附近群组列表明确显示“可直接加入”或“需要邀请码”。
- 完整加入信息允许不包含邀请码。

## 实施

- [x] 协议增加可选的 `inviteRequired` 创建参数和群组摘要字段。
- [x] Broker 默认不生成邀请码，并允许远程成员直接加入开放群组。
- [x] 受限群组继续校验邀请码并保留失败冷却。
- [x] 创建流程默认选择“直接加入”，只有明确选择后才生成邀请码。
- [x] 点击开放群组时直接加入，不显示邀请码输入框。
- [x] 更新群主管理、附近列表和退出提示。
- [x] 更新完整加入信息，使邀请码可选。
- [x] 保持已有邀请码群组的准入规则不变。
