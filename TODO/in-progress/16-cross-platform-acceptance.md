# 阶段 16：跨平台测试与局域网验收

目标：验证 macOS 和 Linux 共用同一套实现，并完成三台设备的真实局域网通信验收。

状态：待实施。

## 已确认边界

- 正式支持平台为 macOS 和 Linux。
- 不兼容 Windows，不在 CI、人工验收和发布声明中承诺 Windows。
- CI 负责平台兼容、进程测试和可用时的本机 mDNS 冒烟测试，真实多设备 mDNS 仍需局域网人工验收。
- 完整群组邀请信息是自动发现不可用时的长期兜底，不是临时测试入口。

### 三台设备真实验收

- [ ] 三台设备连接同一个 Wi-Fi 或有线局域网。
- [ ] 一台设备创建“允许附近设备加入”的群组；用户界面不显示 Broker 或“协作空间”。
- [ ] 另外两台设备在“附近群组”中自动看到该群组名称和在线人数。
- [ ] 两台远程设备输入该群的独立邀请码后加入。
- [ ] 远程成员关闭并重新打开同一 Pi Session 后，使用长期成员凭证自动回群。
- [ ] 验证人对人公开消息。
- [ ] 验证人对 Agent 定向任务。
- [ ] 验证 Agent 对人公开回答。
- [ ] 验证 Agent 对 Agent 自动转交。
- [ ] 验证一台设备断网、重连和恢复原身份。
- [ ] 验证错误邀请码不能看到成员和群聊快照。
- [ ] 验证重新生成邀请码不影响已有成员，新成员不能使用旧邀请码。
- [ ] 验证群主 Session 重开后恢复群主管理，fork/clone 不继承。
- [ ] 验证 VPN 开关不影响群组；切换新普通网络必须重新确认。
- [ ] 验证关闭客户端不会意外启动第二个 Broker。
- [ ] 验证两台本机 Pi 并发首次进入 `/comms` 只启动一个 Broker。
- [ ] 验证 `local` 切换 `lan-host` 后不存在共享同一 SQLite 的第二个 Broker。
- [ ] 验证禁用 mDNS 后可以通过完整群组邀请信息直接加入。
- [ ] 至少完成一个 macOS 与 Linux 混合场景。

- 不做多个 Broker 的数据复制。
- 不做移动端客户端。
