# 阶段 16：跨平台测试与局域网验收

目标：验证 macOS、Linux 和 Windows 共用同一套实现，并完成三台设备的真实局域网通信验收。

状态：待实施。

## 已确认边界

- macOS 和 Linux 作为首批正式支持平台。
- Windows 从本阶段开始进入 CI 和人工验收，未通过前不声明正式支持。
- CI 负责平台兼容和进程测试，真实 mDNS 多播仍需局域网人工验收。
- 手动地址是自动发现不可用时的长期兜底，不是临时测试入口。

## 任务

### 跨平台 CI

- [ ] 将主检查改为 Ubuntu、macOS 和 Windows 矩阵。
- [ ] 三个平台统一使用 Node.js 22 和 `npm ci`。
- [ ] 验证 `better-sqlite3` 可以安装、加载和创建数据库。
- [ ] 验证 Broker launcher 可以启动、握手和关闭。
- [ ] 验证 loopback TCP 的 Broker、Extension 和验收测试。
- [ ] 将依赖审查保留为独立 Linux 任务。
- [ ] 避免在 CI 中依赖 Bash 专用命令和 Unix 信号。

### 平台兼容处理

- [ ] 所有数据目录和入口路径使用 Node 路径 API。
- [ ] 所有临时目录使用系统临时目录。
- [ ] 进程检测和关闭逻辑覆盖 Windows 行为。
- [ ] IPv4、IPv6 和主机名解析失败时给出可读错误。
- [ ] Windows 防火墙提示只建议开放 Private Network。
- [ ] Linux 和 macOS 文档说明常见防火墙与网络隔离问题。

### 自动测试

- [ ] 覆盖三个系统的本机多 Session 通信。
- [ ] 覆盖 TCP 断开、Broker 重启和客户端恢复。
- [ ] 覆盖设备身份、协议版本、心跳和帧限制。
- [ ] 覆盖正确和错误邀请码。
- [ ] 覆盖发现列表、手动地址和连接回退。
- [ ] 保证阶段 0～11 的功能在三个系统上没有回归。

### 三台设备真实验收

- [ ] 三台设备连接同一个 Wi-Fi 或有线局域网。
- [ ] 一台设备创建局域网通信空间并运行唯一 Broker。
- [ ] 另外两台设备自动发现该 Broker。
- [ ] 三台设备输入邀请码后加入同一个群组。
- [ ] 验证人对人公开消息。
- [ ] 验证人对 Agent 定向任务。
- [ ] 验证 Agent 对人公开回答。
- [ ] 验证 Agent 对 Agent 自动转交。
- [ ] 验证一台设备断网、重连和恢复原身份。
- [ ] 验证错误邀请码不能看到群组快照。
- [ ] 验证关闭客户端不会意外启动第二个 Broker。
- [ ] 验证禁用 mDNS 后可以通过手动地址连接。
- [ ] 至少完成一个 macOS 与 Linux 混合场景。
- [ ] Windows 可用设备准备后完成同样验收并记录限制。

## 本阶段不做

- 不做公网服务器或云端账号。
- 不做 NAT 穿透、TURN 或中继。
- 不做跨子网 mDNS 代理。
- 不做多个 Broker 的数据复制。
- 不做移动端客户端。
