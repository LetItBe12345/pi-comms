# 阶段 11：聊天时间线与消息布局

目标：保留独立群聊 ChatView，改用完整追加式时间线和终端原生 scrollback，并修正消息左右布局。

状态：待实施。

## 任务

### 删除自定义滚动

- [ ] 删除 `#scrollFromBottom`。
- [ ] 删除 Ctrl+PageUp 和 Ctrl+PageDown 行为。
- [ ] 删除 `available`、`start`、`end` 和 `maxOffset` 分页计算。
- [ ] 删除 `timeline.slice(start, end)`。
- [ ] 让 `#renderChat()` 返回完整 timeline。

### 改为追加式界面

- [ ] 顶部只保留固定群名，例如 `Pi Comms · 开发组`。
- [ ] 将连接状态、在线人数、Agent 忙碌数、接收权限和待处理数量移到底部。
- [ ] 将成员上线和离线改为追加系统消息。
- [ ] 系统消息居中显示。
- [ ] 避免因顶部动态内容变化而修改已滚出屏幕的行。

### 修正消息方向

- [ ] 当前用户的消息放在右侧。
- [ ] 其他用户的消息放在左侧。
- [ ] 所有 Agent 的消息放在左侧。
- [ ] 当前用户自己的 Agent 仍作为独立成员放在左侧。
- [ ] 反转现有错误方向测试。

### 整块消息对齐

- [ ] 删除当前逐行调用的 `alignRight(line, width)`。
- [ ] 新增 `alignMessageBlock(lines, terminalWidth, blockWidth, side)`。
- [ ] 先按消息块宽度换行，再给整个右侧消息块增加相同缩进。
- [ ] 保证右侧多行消息的左边缘一致。
- [ ] 每行输出前继续校验终端宽度。

### 消息宽度与换行

- [ ] 用户消息最大宽度为终端宽度的 70%。
- [ ] Agent 消息最大宽度为终端宽度的 85%。
- [ ] 窄屏消息最大宽度为终端宽度的 94%。
- [ ] 消息块最小宽度为 12 列。
- [ ] 继续复用 Pi 的 `wrapTextWithAnsi()`、`visibleWidth()` 和 `truncateToWidth()`。
- [ ] 保留用户手动换行和空行。
- [ ] 保留中文、Emoji、组合字符、长 URL 和超长单词的正确换行。
- [ ] Agent 消息复用 Pi 的 Markdown 和 `getMarkdownTheme()`。

### 连续消息合并

- [ ] 同一发送者在两分钟内连续发送时，只在第一条显示名称和时间。
- [ ] 后续连续消息只显示正文。
- [ ] 日期变化后重新显示标题。
- [ ] 系统通知后重新显示标题。
- [ ] 发送者变化后重新显示标题。

### 控制旧消息修改

- [ ] 发送前只在底部显示临时状态 `正在确认消息…`。
- [ ] Broker 确认后再将消息加入正式 timeline。
- [ ] 请求状态只允许显示在最近消息下方。
- [ ] 最终状态确定后不再修改。
- [ ] 尽量不修改已经滚出屏幕的消息。

### 重写测试

- [ ] 测试当前用户在右侧，其他用户在左侧。
- [ ] 测试右侧多行消息的缩进完全一致。
- [ ] 测试所有 Agent 消息都在左侧。
- [ ] 覆盖 40、80 和 120 列窗口。
- [ ] 覆盖中文、Emoji、空行和超长 URL。
- [ ] 验证所有输出行都不超过终端宽度。
- [ ] 验证 timeline 超过终端高度时仍返回全部消息。
- [ ] 删除 Ctrl+PageUp 和 Ctrl+PageDown 测试。
- [ ] 让 `TestTerminal.write()` 记录终端输出。
- [ ] 验证普通新消息不会输出 `\x1b[3J`。
- [ ] 验证输入文字不会触发完整重绘。
- [ ] 验证底部动态状态不会触发完整重绘。
- [ ] 允许窗口尺寸变化触发 Pi 自身的完整重绘。

## 主要改动文件

- `src/tui/chat-view.ts`
- `tests/tui.test.ts`
- `src/extension/index.ts` 仅在必要时做少量调整。

## 本阶段不做

- 不迁移到 Pi 主时间线。
- 不使用 `pi.appendEntry()` 保存群聊消息。
- 不改变 `/comms` 的输入路由、退出方式和 Broker 生命周期。
- 不重新实现 Unicode、Emoji、URL 或 ANSI 换行器。
- 不保证窗口尺寸变化时保留终端 scrollback。
- 不实现鼠标滚动。

## 完成条件

- [ ] `#renderChat()` 始终返回完整 timeline。
- [ ] 终端原生 scrollback 可以查看超出当前窗口高度的历史。
- [ ] 普通追加消息、输入和底部状态变化不清除终端 scrollback。
- [ ] 当前用户消息右侧整块对齐，其他用户和 Agent 消息位于左侧。
- [ ] 连续消息按两分钟规则合并标题。
- [ ] 40、80 和 120 列下所有输出行都不超宽。
- [ ] TUI 自动测试和项目完整检查通过。
