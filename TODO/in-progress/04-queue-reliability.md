# 阶段 4：队列与稳定性

目标：每个 Session 串行处理远程请求，并能应对断线、重启和重复消息。

## 任务

- [ ] 实现每个 Extension 的 `remoteQueue`。
- [ ] Agent 空闲时立即处理远程请求。
- [ ] Agent 忙碌时把请求加入队列。
- [ ] 保证远程请求不抢占当前任务。
- [ ] 保证同一时间只有一个 `activeRequest`。
- [ ] 当前请求回传后自动处理下一条队列。
- [ ] 使用 `requestId` 关联请求和回答。
- [ ] 处理 Socket 断线。
- [ ] 处理 Broker 重启后的重新连接。
- [ ] 防止重复投递消息。
- [ ] 防止重复回传结果。
- [ ] Session 关闭时清理队列和活动请求。

## 完成条件

- [ ] 连续发送多个请求时，B 按顺序逐个处理。
- [ ] 断线重连后不会重复广播已完成请求的结果。
- [ ] Broker 重启后 Extension 可以恢复通信。
