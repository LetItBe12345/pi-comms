# 阶段 9：Agent 对 Agent

目标：Agent 回答可以继续 `@` 另一个 Agent，同时限制自动通信循环。

## 任务

- [ ] 解析 Agent 回答中的 `@Agent名称`。
- [ ] 公开显示触发后续请求的 Agent 回答。
- [ ] 为新通信链生成 `chainId`。
- [ ] 后续请求沿用原 `chainId`。
- [ ] Broker 在每次自动路由时增加轮数。
- [ ] 路由前检查目标是否在线。
- [ ] 路由前检查目标 Agent 权限。
- [ ] 将同一 `chainId` 的自动通信限制为最多 10 轮。
- [ ] 达到上限后停止自动注入。
- [ ] 在群聊中显示已达到轮数上限。
- [ ] 达到上限后等待用户决定是否继续。

## 完成条件

- [ ] 两个 Agent 可以通过公开消息连续自动通信。
- [ ] 所有请求和回答保留正确的 `chainId` 与轮数。
- [ ] 第 10 轮后不会继续自动注入。
