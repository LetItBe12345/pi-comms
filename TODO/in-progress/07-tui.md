# 阶段 7：最小 TUI

目标：提供可以创建或加入群组、查看消息并发送文本的最小聊天界面。

## 任务

- [ ] 创建 `src/tui/chat-view.ts`。
- [ ] 使用 `ctx.ui.custom()`。
- [ ] 使用 `@earendil-works/pi-tui`。
- [ ] 注册正式命令 `/comms`。
- [ ] 进入 `/comms` 时复用当前 Pi Session，不创建新 Session。
- [ ] 提供用户名称和 Agent 名称设置。
- [ ] 提供创建或加入群组入口。
- [ ] 显示群组名称和连接状态。
- [ ] 显示在线用户和 Agent 成员。
- [ ] 从 SQLite 加载并显示公开群聊历史。
- [ ] 显示实时消息。
- [ ] 提供单行输入框。
- [ ] 支持发送普通消息和 `@Agent` 消息。
- [ ] 支持 `@` 自动补全。
- [ ] 支持中文输入法。
- [ ] 支持上下滚动。
- [ ] 支持 Enter 发送。
- [ ] 支持 Esc 退出 TUI。
- [ ] 显示发送失败状态。
- [ ] 显示 Agent 忙碌、空闲和待批准状态。
- [ ] 退出时清理 TUI 状态，但保留当前 Pi Session 上下文。

## 本阶段不做

- 不实现鼠标操作。
- 不实现复杂富文本。

## 完成条件

- [ ] 两个终端能通过 TUI 完成公开文本聊天。
- [ ] 两边都能看到群名、在线成员、历史消息和实时消息。
- [ ] `@Agent` 请求可完成既有的远程 Agent 闭环。
