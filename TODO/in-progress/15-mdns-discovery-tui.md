# 阶段 15：mDNS 自动发现、附近群组与群主管理

目标：让用户在 `/comms` 中直接看到、创建和加入群组，并补齐群主、长期成员、按群邀请、后台可用性和网络隐私。Broker、端点和 mDNS 只作为后台实现。

状态：实施中。核心协议、数据库迁移、每群邀请、长期成员、发现目录、
群主管理、VPN 网卡过滤、生命周期和跨平台测试已落地。剩余工作主要是三分区
首屏、普通网络切换确认、完整后台恢复、版本冲突处理和真实网络验收。

## 已确认产品模型

- 一个数据目录只有一个 Broker 和一份 SQLite，一个 Broker 可以承载多个群组。
- 用户只选择和管理群组，不选择或管理 Broker，也不看到“协作空间”。
- 群组只能在自己的设备上创建；远程用户只能加入，不能在别人的 Broker 上创建群组。
- 每个群组有独立群主、独立邀请码、长期成员、可见范围和阻止列表。
- 群主绑定创建群组的本机 Pi Session，不支持群主转让。
- 同一个 Pi Session 可以担任多个群组的群主，但同一时间只能进入一个群聊。
- Broker 生命周期由所有本机群组和在线成员的状态自动决定，不提供单独的 Broker 管理界面。
- “附近可见”只公开群组名称和在线人数；不公开成员、Agent、消息和聊天记录。

## 任务

### 重构群组持久化模型

- [x] 为群组持久化群主凭证摘要、可见范围、当前群组邀请码、后台保持、自启设置和创建时间。
- [x] 为长期成员持久化群组 ID、稳定 Session 身份、成员凭证摘要、用户名称、Agent 名称、最后活跃时间和移出状态。
- [x] 群主凭证与创建群组的本机 Pi Session 绑定；同一 Session 恢复时重新显示群主管理入口。
- [x] `/fork` 和 `/clone` 创建的新 Session 不继承群主凭证。
- [x] 提供仅限本机的“恢复群主管理权”；恢复后旧群主凭证立即失效，不能同时存在两个群主。
- [x] 同一设备内群组名称唯一，英文大小写不敏感；不同设备允许同名群组。
- [x] 群组改名不改变 `groupId`、邀请码、成员凭证和聊天记录，并追加公开系统消息。
- [ ] 解散群组时永久删除该群的邀请、成员、阻止记录、消息和 Agent 请求，不影响其他群组。
- [ ] 解散操作要求输入群组名称并二次确认；离线成员下次连接时收到一次“群组已解散”并清理本地凭证。

> 已完成数据库级解散和本机二次确认；离线成员的一次性解散通知仍待补齐。

### 实现按群邀请与长期成员

- [x] 删除共享邀请码逻辑，邀请码只属于单个群组。
- [x] 每个附近群组使用独立 10 位 Base32 邀请码，显示时按 `5-5` 分组。
- [x] 完整邀请信息同时携带端点和群组邀请码；粘贴后直接定位群组，不再选择设备或群组。
- [x] 邀请码只用于首次加入；成功后签发独立长期成员凭证。
- [ ] 断线、Session 重开、Broker 重启和网络切换后使用长期成员凭证自动恢复，不重复输入邀请码。
- [x] 重新生成邀请码只影响新成员，不断开或废除已有长期成员。
- [x] 关闭附近加入时废除当前邀请码；再次开放时生成新邀请码。
- [x] 邀请码不自动过期，直到重新生成、关闭附近加入或解散群组。
- [x] 同一来源 60 秒内连续输错 5 次后冷却 30 秒；成功后清零。
- [x] 错误文本、普通日志和 mDNS 元数据不得包含邀请码。
- [x] 正确邀请码直接加入，不增加群主审批队列。

> 长期凭证已覆盖断线、Session 重开和 Broker 重启；普通网络切换后的端点重发现仍待完成。

### 实现成员生命周期与移出

- [x] 关闭群聊或关闭 Pi Session 只让成员离线，不删除长期成员身份。
- [x] “退出群组”才删除长期成员凭证；以后需要新邀请码才能再次加入。
- [x] 群主不能退出自己的群组，只能解散群组。
- [x] 长期成员离线后保留用户名称和 Agent 名称，避免被新成员占用。
- [ ] Pi Session 保存默认用户名称和 Agent 名称；加入群组时自动填入，冲突时才要求修改。
- [x] 每个群组保存名称覆盖，重新连接时恢复该群上次使用的名称。
- [x] 群主可以移出在线或离线成员；一次操作同时移出该用户和其 Agent。
- [x] 移出按稳定 Pi Session 身份写入群组阻止列表，不按用户名或 IP。
- [x] 被移出成员停止重连、从“已加入”移除并清理成员凭证。
- [x] 群主解除阻止后，对方仍需使用当前邀请码重新加入。
- [ ] 成员管理只显示名称、Agent 名称、在线状态和最后活跃时间，不显示 IP、设备 ID 或 Session ID。

> 成员面板已隐藏技术身份；Agent 名称和最后活跃时间的完整展示仍待补齐。

### 封装 mDNS 发现

- [x] 增加独立 `BrokerDiscovery` 接口，业务层不直接依赖具体 mDNS 库。
- [x] 使用 `bonjour-service` 发布和浏览 `_pi-comms._tcp.local`。
- [x] 定义内部 `DiscoveredBroker`，包含 `brokerId`、主机、地址列表、端口、协议版本和应用版本。
- [x] 定义 TUI 使用的 `NearbyGroup`，包含 `brokerId`、`groupId`、群组名称、在线人数、端点和连接状态。
- [x] 使用 `brokerId + groupId` 作为附近群组唯一身份，不使用显示名称去重。
- [x] TXT 只发布 `txtvers`、协议版本、`brokerId` 和应用版本。
- [x] TXT 不发布邀请码、群组目录、成员、消息或 Session 信息。
- [x] Broker 完成 TCP 监听后发布，关闭前撤销；浏览停止后释放浏览器、定时器和 UDP Socket。
- [x] 提供可替换的假发现实现，供 TUI 和单元测试使用。

### 查询并汇总附近群组

- [x] 增加只读群组目录协议，在加入前返回群组 ID、名称和在线人数。
- [x] 目录查询不创建客户端身份，不返回成员和消息，响应后关闭短连接。
- [x] 目录只返回允许附近加入的群组，不提供隐藏但可邀请加入的第三种模式。
- [x] 在线人数按 Pi Session 计数，用户和其 Agent 不重复计数。
- [ ] 持续处理服务上线、下线、地址更新、TTL 失效、群组改名和在线人数变化。
- [x] 多个 Broker 的结果合并为一个附近群组列表；不同 Broker 的同名群组并列显示，必要时补充设备名称。
- [x] 过滤 loopback、未指定地址、`0.0.0.0` 和 `::`；优先可用私网 IPv4，再回退 IPv6。
- [ ] 每个候选地址使用短连接超时；IPv6 链路本地地址保留 scope ID。
- [x] TCP 握手校验 `brokerId` 和协议版本；过期 IP 指向其他 Broker 时拒绝。
- [ ] 协议不兼容时显示“发现 1 台设备需要更新 Pi Comms”，不展示 Broker 术语。

> 当前附近发现是一次性聚合；持续刷新、TTL 过期和逐候选地址回退仍待完成。

### 处理网络与 VPN

- [x] 自动跟随操作系统主要的非 VPN 网络，不让用户选择网卡。
- [x] 自动忽略常见 `tun`、`tap`、`utun` 和 `wg` VPN 接口。
- [x] VPN 打开或关闭不视为切换网络，不暂停群组、不关闭 Broker，也不重新要求授权。
- [ ] mDNS 和局域网 TCP 使用已确认的普通网络，互联网模型请求继续按系统路由使用 VPN。
- [ ] 切换到新的普通网络时暂停所有附近群组广播和新连接，只询问“是否允许附近设备看到你的群组”。
- [ ] 记住最近确认的普通网络；回到该网络时自动恢复附近群组和成员重连。
- [ ] 登录后自动开放不能绕过新网络确认。
- [x] VPN 禁止局域网访问时提示检查“允许局域网访问”，不要求关闭 VPN。

### 管理 Broker 生命周期

- [x] Session 关闭只断开该客户端，不直接关闭 Broker。
- [x] 只要仍有本机或远程成员在线，Broker 继续运行。
- [x] 最后一名成员离线后等待 5 分钟；没有后台要求时关闭附近服务。
- [x] 群组提供“无人在线时仍允许附近加入”，默认关闭。
- [x] 群组提供依赖于上一项的“登录后自动开放”，默认关闭。
- [x] 关闭“无人在线时仍允许附近加入”时同步关闭“登录后自动开放”。
- [x] 只要任意群组需要后台可加入，Broker 保持运行；所有群组都不需要时允许空闲退出。
- [x] 只要任意群组开启登录后自动开放，注册用户级自启；全部关闭后移除。
- [x] Ubuntu 使用用户级 systemd，macOS 使用 LaunchAgent，不要求管理员密码。
- [ ] 首次注册自启前说明将修改当前用户的后台启动配置并请求确认。
- [ ] Broker 崩溃前有在线成员或后台开放群组时自动恢复；完全空闲且未开启后台时不自动重启。
- [x] 恢复后从 SQLite 加载群组并恢复 mDNS，长期成员自动重连。
- [ ] 本机 Extension 发现旧发行版 Broker 时，使用当前发行版优雅重启并更新自启入口。
- [ ] 不允许开发版 Extension 静默接管发行版 Broker。

## 本阶段不做

- 不发现或直连单个 Pi Session。
- 不在 mDNS TXT 中发布动态群组目录。
- 不做 Broker 自动选举或多个 Broker 的 SQLite 同步。
- 不跨 VLAN、子网或公网发现。
- 不做群主转让、远程 Broker 管理或账号系统。
- 不做隐藏但可凭邀请加入的第三种群组模式。
- 不做一次性、定时过期邀请码或加入审批队列。
- 不做账号级封禁；新建 Pi Session 仍可能绕过按 Session 的移出限制。
- 不兼容 Windows。
