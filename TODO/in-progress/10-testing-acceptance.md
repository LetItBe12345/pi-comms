## 首个验收场景

分别在两个终端启动 Pi 并加载 Extension。A 发送：

```text
@B-Pi 检查当前项目的 package.json
```

- [ ] A 和 B 都能看到公开消息。
- [ ] 消息只进入 B 的 Pi Session。
- [ ] B 忙碌时消息进入队列。
- [ ] B 的最终回答只回传一次。
- [ ] 回答能通过 `requestId` 对应原始请求。
- [ ] A 能看到 B 的回答。
- [ ] B 退出后，B 用户和 B-Pi 同时离线。
- [ ] 离线目标不能收到消息。
- [ ] 失败状态会公开显示。

## MVP 完成定义

- [ ] 两个 Pi Session 可以稳定加入同一群组。
- [ ] 用户和 Agent 都显示为独立成员。
- [ ] 同一群内所有显示名称唯一。
- [ ] 普通消息公开广播。
- [ ] `@Agent` 消息只注入目标 Session。
- [ ] Agent 最终回答公开且只回传一次。
- [ ] Agent 忙碌时按顺序处理。
- [ ] 三种接收权限可用。
- [ ] Agent 对 Agent 最多自动通信 10 轮。
- [ ] Session 生命周期和在线状态正确。
- [ ] 私人 Pi 对话不会泄露到群聊。
