# 局域网故障排查

## 看不到附近群组

1. 确认两台设备连接同一个普通 Wi-Fi 或有线网络。
2. 确认群主已经允许附近设备加入。
3. 公共 Wi-Fi、访客网络和部分网络可能开启“客户端隔离”，设备之间无法直连。
4. 暂时关闭系统防火墙做对比测试；确认原因后只为 Pi/Node.js 开放本地网络，
   不要长期关闭防火墙。
5. 自动发现仍不可用时，使用群主界面提供的完整群组加入信息。

## VPN 开启后断开

Pi Comms 会忽略常见 VPN 网卡，继续选择普通网络。部分 VPN 的 Kill Switch 或
“禁止局域网访问”会在系统层阻止本地流量，此时需要在 VPN 设置中允许 LAN/
Local Network。Pi Comms 不绕过 VPN 主动设置的封锁。

## macOS

- 第一次访问本地网络时，允许终端或 Pi 使用本地网络。
- 在“系统设置 → 网络 → 防火墙”中确认终端、Node.js 或 Pi 没有被阻止。
- 如果刚修改权限，完全退出旧 Pi Session 后重新打开。

## Ubuntu/Linux

- 检查当前网络不是访客网络。
- 使用 UFW 时，确认同一私网可以访问 Pi Comms 的 TCP 端口和 UDP 5353。
- 用户级自启依赖 systemd user service；无图形登录或禁用 user service 的环境需要
  先修复系统登录服务。

## 邀请信息仍然连不上

- 当前版本只支持普通网络 IPv4。
- IPv6 会提示“当前版本暂不支持 IPv6”。
- 主机名会提示“当前版本请使用 IPv4 地址”。
- 不支持跨子网、公网、NAT 穿透、TURN、中继或多个 Broker 的数据同步。
