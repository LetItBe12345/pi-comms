# 跨设备身份与连接测试

## 准备

- 两台电脑安装同一版本 Pi Comms。
- 一台启动局域网 Broker，两台都连接它。
- 两台电脑各启动一个 Pi，并进入 `/comms`。

## 检查

1. 两台 Pi 使用相同 `sessionId` 测试值，确认都能独立在线。
2. 创建并加入同一群组，确认四个成员正常显示。
3. 临时断开其中一台网络，在 15 秒内恢复，确认成员 ID 和群组不变。
4. 断网超过失联期，确认成员被移除；恢复后需要重新加入群组。
5. 重启 Broker，确认仍运行的 Pi 自动重新加入并加载历史。
6. 创建暂停的 Agent 通信链，确认只有原设备和 Session 可以继续或结束。

## 通过标准

- 相同 `sessionId` 不会互相覆盖。
- 短暂断网可以恢复，长期断网会正式离线。
- 其他设备不能接管连接或操作暂停链。
